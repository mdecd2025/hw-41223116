五、Camera 裝置可以調整哪些參數？

可在機器人設定中調整以下參數：

參數名稱 功能
width, height 攝影機解析度
fieldOfView 視野角度（FOV）
near, far 近 / 遠裁剪面（剪除畫面以外區域）
motionBlur 是否模擬動態模糊
antiAliasing 抗鋸齒處理（改善畫質）

六、影像解析度的設定會對模擬有什麼影響？
解析度高 解析度低
更清晰、細節更多 畫質粗糙
影像處理更準確 難以辨識小物體
會增加計算量與模擬延遲 效能較佳，適合快速模擬

建議依照應用需求（例如只辨認顏色不需高解析）選擇合適解析度。


七、如何將攝影機影像儲存成圖片檔？

可使用 Webots 提供的方法：

python
camera.saveImage("image.png", quality)

  • quality：介於 1 到 100，代表影像壓縮品質。

完整例子：

python
camera.saveImage("snapshot.png", 100)
camera.saveImage("snapshot.png", 100)

也可結合 OpenCV 寫入圖片：

python
import numpy as np
import cv2

image = camera.getImage()
width = camera.getWidth()
height = camera.getHeight()
img_array = np.frombuffer(image, np.uint8).reshape((height, width, 4))
bgr_image = cv2.cvtColor(img_array, cv2.COLOR_BGRA2BGR)
cv2.imwrite("snapshot_cv.png", bgr_image)

八、若攝影機影像顯示異常，可能的原因有哪些？
問題情形 可能原因
畫面為黑（全 0 值） 沒有 enable() 或 timestep 設為 0
畫面畫質太差或閃爍 解析度太低 / 頻率過低 / 模擬效能不足
顏色異常（如全綠） RGB 通道讀取錯誤，需確認 imageGetRed() 等使用正確
儲存圖片為空檔 saveImage() 檔名路徑錯誤或沒寫入權限
調用 OpenCV 時轉換錯誤 需先轉換成 NumPy 格式、注意通道順序與格式
